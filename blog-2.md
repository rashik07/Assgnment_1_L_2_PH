# How `Pick` and `Omit` Keep Your TypeScript Code DRY

## Introduction

When building applications, we often need different versions of the same data. For example, a `user` object might contain many properties, but some pages or APIs only need a few of them. Writing separate interfaces for each case leads to duplicated code and extra maintenance.

TypeScript's `pick` and `omit` utility types help solve this problem by creating new types from existing ones, keeping your code `DRY`

## Example 

```typescript
interface User {
  id: string;
  name: string;
  email: string;
  passwordHash: string;
  role: "admin" | "editor" | "viewer";
  createdAt: Date;
  updatedAt: Date;
}
```

## Using `Pick`

`Pick` selected only the properties you need.

```typescript
type UserSummary = Pick<User, "id" | "name" | "role">;

```

Result:

```typescript
{
  id: string;
  name: string;
  role: "admin" | "user";
}
```

This is useful when displaying a user list or profile card.

## Using `Omit`
`Omit` removes properties you don't want.

```typescript
type PublicUser = Omit<User, "passwordHash">;
```
This creates a user type that can be safely returned from an API without exposing sensitive information.


## Real-World Usage

```typescript
 type CreateUserPayload = Omit<
  User,
  "id" | "createdAt"
>;

type UpdateUserPayload = Partial<CreateUserPayload>;
```
`CreateUserPayload` excludes fields generated by the server.
`UpdateUserPayload` makes all fields optional for updates
